# Circuit catalog

All circuits are OpenQASM 2.0 (`circuits/*.qasm`).

## Families (high level)

- **GHZ** (easy/structured, scales to large N)
- **GraphState**
- **Deutsch–Jozsa**
- **QFT / QFT_Entangled**
- **QPE_Exact**
- **Grover (no ancilla / v-chain)**
- **QAOA / VQE / Portfolio variants**
- **QNN**
- **W-State**
- **TwoLocalRandom** (entanglement stressor)
- **Amplitude Estimation**
- **Ground State**
- **CutBell** (custom-generated)

## Sources

Most circuits are from:
- **MQT Bench** (Munich Quantum Toolkit): https://github.com/cda-tum/mqt-bench

Custom circuits:
- **CutBell** is generated by scripts in this repo (see `scripts/`).

## Suggested train/test split (24/12)

For local model development, we recommend splitting **by circuit file** (not by individual
`results[]` rows). In other words: all `(backend, precision)` entries for a given `file`
should stay together in either train or test.

We also recommend keeping **small / medium / large** circuits in both sets.

### Size buckets (based on `n_qubits`)
- Small: `n_qubits ≤ 20`
- Medium: `21 ≤ n_qubits ≤ 60` (in this repo, most `n=30` circuits)
- Large: `n_qubits ≥ 100` (here: 100 and 130)
